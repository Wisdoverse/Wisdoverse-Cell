# Support

## Public Questions

Use GitHub issues for reproducible bugs, documentation problems, and focused
feature requests. Include the version, deployment mode, logs, and minimal
reproduction steps when possible.

## Security Reports

Do not open public issues for vulnerabilities. Follow the responsible disclosure
process in [SECURITY.md](./SECURITY.md).

## Commercial Use

License 1.1. Commercial production use, SaaS, hosted services, managed services,
resale, sublicensing, and competing products require a separate commercial
