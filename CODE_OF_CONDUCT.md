# Code of Conduct

## Our Pledge

The Wisdoverse Cell project is committed to providing a welcoming and
professional environment for contributors, evaluators, and users working on
AI-native operating systems and agent orchestration.

## Our Standard

This project has adopted the
[Contributor Covenant, version 2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/)
as its code of conduct. The full text of the Covenant defines expected and
unacceptable behavior, enforcement responsibilities, scope, and enforcement
guidelines, and is incorporated here by reference.

## Project-Specific Expectations

In addition to the Covenant, contributors to this repository should:

- Keep technical discussions specific to code, documentation, design, security,
  or product behavior.
- Treat public issues and pull requests as part of the permanent project
  record.
- Disclose conflicts of interest when they affect a contribution or review.
- Never publish private user data, customer identifiers, or unredacted logs in
  public issues or pull requests.

## Reporting

Conduct concerns may be reported privately to the maintainer team at
**dev@wisdoverse.com**. Reports are reviewed by maintainers not involved in
the incident, and the reporter is informed of the resolution.

Security vulnerabilities must be reported through [SECURITY.md](./SECURITY.md),
not through the conduct channel and not through public issues.

## Enforcement

Maintainers may, in response to violations of the Covenant or the
project-specific expectations above:

- Edit, hide, or remove comments, commits, code, or other contributions.
- Close issues or pull requests.
- Temporarily restrict participation.
- Permanently block accounts that demonstrate sustained or severe
  violations.

Enforcement decisions follow the Covenant's enforcement guidelines and are
applied consistently across contributors.

## Scope

This Code of Conduct applies within all project spaces, including the GitHub
repository, issues, pull requests, code reviews, project documentation, and
any community space where an individual is representing the project. It also
applies in private channels when those interactions affect project work.
